# createRef & ref

# 介绍

  • string ref将会被废弃，不推荐使用
  • function ref（接受参数，组建的实例、dom节点的实例）
    • ref传入一个方法，这个方法就是节点对应的实例，如果是组建就是组建的实例，如果是dom节点就是dom节点的实例。
  • create ref
    • ref接受一个对象。对象通过React.createRef()这个api创建。

# createRef api

前面两种在源码里面都没有体现，createRef在源码中的ReactCreateRef.js。 ref是为了获取某哥节点的实例的，如果组建是个function component，他就是个pure component，没有this，通过ref拿不到，就会报错了。

# forwordRef

function component是没有this的，通过ref传递之后拿不到东西，有时候我们需要用到别人的开源组建，不知道是不是pure..也不知道是不是connect的组建（返回了包装的组建），我们要获取我们自己写的那个组建的实例而不是HOC包装过的时候，我们正常是不可以在下一层组建里面拿到上一层的属性的， 拿到ref，可以传递给我们需要包装的组建（HOC），这样不会违背开发者的意图。

import React from 'react'

const TargetComponent = React.forwardRef((props),ref)=>(
    <input type="text" ref={ref}>
))

export default class Comp extends React.Component{
    constructor(props){
        super(props);
        this.ref=React.createRef();
    }
    ComponentDidMount(){
        this.ref.current.value='ref get input';
    }
    render(){
        return <TargetComponent ref={this.ref}/>
    }
}
