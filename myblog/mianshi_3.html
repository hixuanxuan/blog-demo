# 虚拟dom virtual dom

  • vdom是vue和react的核心

  • vdom是什么，为什么会出现vdom 浏览器本身创建的dom结点属性非常多，非常复杂，尽量少的进行dom操作，用js代替dom操作的执行，dom的操作是昂贵的，js的运行效率高。 尽量减少dom的操作，而不是推倒重新来（容器清空，塞入新的东西） 项目越复杂，影响就越严重 vdom可以解决上面的问他

vdom是虚拟dom，virtual dom 用js模拟dom结构 dom操作非常昂贵 将dom对比操作（算法）放在js层，提高效率

  • vdom如何应用，核心api是什么 以snabbdom虚拟dom库来举例子 h函数 h('标签名',{...属性...},[...子元素...]) //数组里面的元素也是h(..) h('标签名',{...属性...},"子元素"); //说明子元素是文字结点 patch(container,vnode);//首次渲染的时候将虚拟dom渲染到容器 patch(vnode,newVnode);//再次渲染的时候比对结点

  • 介绍一下diff算法 什么是diff算法 去繁就简

git diff xxx 可以查看修改前和修改后的不同 在线diff的对比器

  • vdom为何使用diff算法 diff算法非常复杂，源码量很大，去繁就简，找出核心原理
  • dom操作是昂贵的，尽量减少dom操作。
  • 找出本次DOM必须更新的节点来更新，其他的不更新
  • 这个“找出”的过程，就需要diff算法。
  • diff算法的实现流程

核心实现的函数 patch(container,vnode);//render patch(vnode,newVnode);//re-render

    function updateChildren(vnode,newVnode){
        var children=vnode.children||[];

        //如果子元素一样多，怎样对比子元素
        children.forEach((childVnode,index)=>{

    //渲染函数
    var vnode;
    function render(data){
        var newVnode=h('table',{},data.map((item)=>{
            var tds=[];
            for(let i in item){
                //看看i是不是本身自由的属性，而不是继承（原型中）的属性
                if(item.hasOwnProperty(i)){
                    tds.push(h('td',{},`${item[i]}`))
                }
            }
            //行的子元素是列
            return h('tr',{},tds);
        }));
        if(vnode){
            patch(vnode,newVnode);
        }else{
            //初次渲染
            patch(container,newVnode);
        }
        //存储上次vnode的结果
        vnode=newVnode;
    }
    //初次渲染
    render(data);
    btn.addEventListener('click',function(){
        data[1].age=30,
        data[2].address="wcnm"
        //re-render
        render(data);
    })
