# 单例模式

# 特点

  • 系统中唯一被使用的模式
  • 一个类只有一个实例

# 示例

  • 登录框
  • 购物车
  • jquery的$
  • 弹窗的显示与隐藏
  • vuex redux的store或者state
//直接写在class里面的函数会new一个对象就有一个，而通过obj.func这样的函数是静态函数，不管对象多少个实例都只有一个
class SingleObj{
    login() {
        console.log('login...');
    }
}

SingleObj.getInstance = (function() {
    let instance;
    return function () {
        if (!instance) {
         instance = new SingleObj();
        } 
         return instance;
    }
})();
//封装成立即执行的函数

// 注意 js和java不一样，java可以让构造函数私有化，
//这个类变成内部类无法再外部new，js的类没有私有类，只能通过文档约束这个类不能在外部new
let obj1 = SingleObj.getInstance();
obj1.login();
let obj2 = SingleObj.getInstance();
obj2.login();

//单例模式就是我们每次获取的都是一个东西，
console.log(`ob1===obj2: ${obj1 === obj2}`);
//注意等号优先级和加号优先级....

打印结果是true，说明是同一个实例，符合单例模式。不能避免的就是私有化构造函数。

# 设计原则验证

  • 符合单一职责原则，只能实例化唯一对象
  • 没法具体开放封闭原则，但是不违背开放封闭原则。
