# 类型转换

显示类型转换 Number函数 原始类型转换

  • 数值：转换后还是原来的值。
  • 字符串：如果可以被解析为数值，则转换为相应的数值，否则得到NaN。空字符串转换为0.
  • 布尔值：true转化为1，false转换为0.
  • undefined:转换成NaN
  • null:转换成0 （比parseInt还要严格） 对象类型的转换
  • 先调研对象自身的valueOf方法，如果该方法返回原始值（数值、字符串和布尔值），则直接对该值使用Number方法，不再进行后续步骤。
  • 如果valueOf方法返回复合类型的值，再调用对象自身的toString方法，如果toString方法返回原始类型呃值，则对该值使用Nuber方法，不再进行后续步骤。如果toString方法返回的是复合类型的值，则报错。

显示类型转换 String函数 原始类型转换

  • 数值：转换为相应的字符串
  • 字符串：转换后还是原来的值
  • 布尔值，true转为"true",false转换为“false”
  • undefined:转为"undefined"
  • null:转为"null"

# HTTP协议

  • HTTP协议的主要特点

    • 简单快速 客户向服务器请求服务时，只需传送请求方法和路径。请求方法常用的有GET、HEAD、POST。每种方法规定了客户与服务器联系的类型不同。由于HTTP协议简单，使得HTTP服务器的程序规模小，因而通信速度很快。
    • 灵活 可以完成不同数据类型的传输
    • 无连接 限制每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断开连接。
    • 无状态 客户端和服务端两种身份，端开了后再次链接服务端是无法确定身份。

  • HTTP报文的组成部分

    • http请求报文HTTP请求报文主要包括请求行、请求头部（HOST下面的）以及请求的数据（实体）三部分
    • HTTP响应报文同样也分为三部分，有状态行、首部行、实体

  • HTTP方法（很基础）

  • POST和GET区别 下面的点比较多

    • GET在浏览器的回退是无害的，但POST会再次请求
    • GET产生的URL地址可以被收藏，而POST不可以
    • GET请求会被浏览器主动缓存，而POST不会，除非手动设置
    • GET请求的参数会被完整地保留在浏览器的历史记录里面，而POST中的参数不会保留。
    • GET请求的URL长度有限制，post长度没有限制（？？？其实也是有限制的）
    • 对参数的数据类型，GET只接受ASC2字符，而post没有限制
    • GET参数直接暴露url，比较不安全
    • GET参数通过URL传递，POST放在Request Body中。

  • HTTP状态码

    • 1.x 指示信息，
    • 2.x成功
    • 3.x重定向
    • 4.x客户端错误
    • 5.x服务端错误 206 Partial Content：客户端发送了带有range头的get请求，服务器完成了他（audio video播放了很大的东西） 301 所请求的页面已经转移到新的url302 区分上面，是临时重定向 304 NOT Modified：客户端有缓冲的文档并发出了一个条件性的请求，服务器告诉客户，原来的缓冲文档还可以继续使用。 403 资源禁止被访问

  • 什么是持久连接 回答方向1） 1.1版本开始支持 （2）持久链接与非持久链接的区别

  • HTTP协议采用“请求-应答”模式，当使用普通模式，即非Keep-alive模式，每个请求/应答客户和服务器都要新建一个链接，完成后断开

  • 当使用持久连接Keep-Alive时，客户端到服务器的连接持久有效，当出现对服务器的后继请求时候，Keep-Alive功能避免了建立或者重新连接。

  • 什么事管线化 请求1=》响应1=》请求2=》响应2=》请求3=》相应3 变成了下面的 请求1=》轻轻2=》请求3=》响应1=》响应2=》响应3 有一种打包了的感觉。 是在服务端如何实施的，背后的原理是什么？ 管线化机制通过持久化完成，HTTP1.1+ GET和HEAD请求可以，POST有所限制 管线化不会影响响应到来的顺序，响应返回的顺序没有改变。 初次创建连接不应启动管线机制，因为对方（服务器）不一定支持HTTP1.1以上的协议

直接把方法写在构造函数里面，那么每一个实例都会拷贝这么一个方法占用内存，如果写到原型链上，实例可以找到这个方法，但又不会拷贝他占用内存，查找原型链上的方法的步骤由JS引擎完成。

# new运算符的原理

原型链类 new运算符

  • 新对象创建，继承自foo.protoType
  • 构造函数被执行，执行的时候，相应的传参会被传入，同时上下文（this）会被指定为这个新实例。new foo等同于new foo(),只能用在不传递任何参数的情况。
  • 如果构造函数返回了一个对象，那么这个对象会取代整个new出来的结果。如果构造函数没有返回对象，那么new出来的结果为步骤1创建的对象。
// 创建构造函数，构造函数关联对象
// 执行构造函数
// 判断构造函数的返回结果是不是对象类型，如果是，返回该结果，否则，返回第一步创建的对象
var mynew=function(func){
    var o=Object.create(func.protoType);
    var k=func.call(o);
    if(typeof k==='object'){
        return k;
    }else{
        return o;
    }
}

# 类与继承

  • 如何实现继承
  1. 子类里面执行父级的构造函数
function Parent1(){this.name='parent'}
function Child1(){
    // 执行父亲的构造函数,修改this指向，然后里面的属性都会挂载到Child的上面
    Parent1.call(this);
    this.type='child';
}
// 缺点：构造函数也有自己的原型链，没有把原型链的东西继承过来。
  1. 原型链实现继承
function Parent2(){
    this.name='parent2';
}
function Child2(){
    this.type='child2';
}
// 关键步骤
Child2.prototype=new Parent2();
// let child2 = new Child2();
// child2.__proto__就是Child2.protoType也就是指向了Parent2的实例，上面有着Parent2的属性

缺点：一个类实例了多个属性，但是改了一个对象，另一个对象也改了，原因：原型链上属性是实例所公用的。

  1. 组合方式
function Parent3(){this.name='parent'}
function Child3(){
    // 执行父亲的构造函数,修改this指向，然后里面的属性都会挂载到Child的上面
    Parent3.call(this);
    this.type='child';
}
// Child3.prototype=new Parent3();
Child3.prototype=Parent3.prototype;
// 问题来了，用Child3的实例的constructor会发现是Parent3的构造函数，所以他自己是没有constructor的

优化方案

function Parent4(){this.name='parent'}
function Child4(){
    // 执行父亲的构造函数,修改this指向，然后里面的属性都会挂载到Child的上面
    Parent4.call(this);
    this.type='child';
}
// Child3.prototype=new Parent3();
Child4.prototype=Object.create(Parent4.prototype);
// 起到了隔离的作用，不共享原型链
Child4.prototype.constructor=Child4;

不仅有了中间原型对象，还手动添加了constructor。解决了上面所有问题，最后一种才是最完美的继承方式。
