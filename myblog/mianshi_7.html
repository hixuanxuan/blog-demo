# Hybrid

# 1. file协议与http协议区别

  • file协议
    • 其实在一开始接触开发，就已经使用了file协议

# 1.1 格式不同

file协议 file:// /file path

# 1.2 加载速度

file协议非常快，http协议需要网络请求，比较慢

file协议用于访问本地计算机的文件，它是针对本地的，它只是简单的请求了本地文件，将其作为一个服务器未解析的静态文件打开，不可进行跨域操作。file协议有一个类似http的远程访问，即ftp协议（文本传输协议）。

http访问本地html文件，相当于将本机作为一台http服务器，通过localhost访问本地的服务器，然后通过http服务器去访问本机文件资源。本地搭建http服务器开放端口后，别人可通过http访问到你电脑中的文件

# 2.Hybird常用

# 2.1 Hybird是什么，为何用hybird

不是所有场景都适合Hybird：

  • 使用Native ： 体验要求极致，变化不频繁，如头条的首页
  • 使用Hybird ： 体验要求高，变化频繁，如头条的新闻详情页面
  • 使用h5 : 体验无要求，不是很常用，如举报、反馈、等页面

实现：

  • 前端做好静态页面（html,js ,css），将文件转交给客户端，
  • 客户端拿到静态页面，以文件形式存储到app中
  • 客户端在一个webview中
  • 使用file协议加载静态页面

遗留问题

  • app发布之后，静态文件如何更新
  • 静态文件如何获取内容

# 2.2 介绍一下hybird更新和上线的流程

思考（目的，可行途径）

  • 要替换每个客户端的静态文件
  • 只能客户端来做
  • 客户端去server下载最新的静态文件
  • 我们来维护server的文件

完整流程 zip包 （时间戳版本管理）（html,css,js）=》客户端解压=》

  • 分版本 有版本号
  • 将静态文件压缩成zip包，上传到服务端
  • 客户端每次启动，都去服务端检查版本号。
  • 如果服务端版本号，就去下载最新的zip包
  • 下载完成后解压包，将现有文件覆盖

# 2.3 Hybird和h5的区别

  1. 优点 体验更好，和Native基本一致
  2. 缺点
  • 开发成本高。联调、测试、查bug都比较麻烦。
  • 运维成本高
  1. 适用的场景
  • hybird：产品的稳定的功能，体验要求高，迭代频繁
  • h5:单次的运营活动（如XXX红包）或者不常用功能

# 3. JS和客户端通讯

# 3.1. 基本形式

  • 封装方法，比如invoke.js，内置到客户端
  • 客户端每次启动webview，都默认执行invoke.js ，将方法挂载到了window上面
  • 本地加载，免去网络加载的时间，更快。
  • 本地加载没有网络请求，黑客看不到schema协议，更安全

# 3.2. schema协议的简介和使用

  • 通讯的基本形式，（下面附上代码）调用能力，传递参数，监听回调
  • 对schema协议的理解和使用
  • 调用schema代码的封装
  • 内置上线的好处，更快，更安全

# 3.3. schema使用的封装

    function _invoke(action,data,callback){
        // 拼接schema协议
        var schema='mayapp://utils';
        schema+='/'+action;
        schema+='?a=a';
        var key;
        for(key in data){
            if(key in data){
                if(data.hasOwnProperty){
                    schema+='&'+key+'='+data[key];
                }
            }   
        }
        // 处理callback
        var callbackName='';
        if(typeof callback==='string'){
            callbackName=callback;
        }else{
            callbackName=action+Date.now();
            window[callbackName]=callback;
        }
        schema+='&callback='+callbackName;
        // 触发
        var iframe=document.createElement('iframe');
        iframe.style.display='none';
        iframe.src=schema;//发出一个请求被app拦截
        var body=document.body;
        body.appendChild(iframe);
        setTimeout(function(){
            body.removeChild(iframe);
            iframe=null;
        })
    }
