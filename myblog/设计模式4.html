# 工厂模式

# 使用场景

  • JQuery $('div') $('#id')
  • React.createElement
  • Vue异步组件

  1. jquery如果没有无new操作，缺点将会是？ 第一：书写麻烦。JQuery的链式操作将会变成噩梦。 第二：构造函数的名字发生变化，即jQuery发生变化，将会是灾难性的。 工厂把真正的构造函数和使用者分离开，创建时候有了统一的入口，不能让别人自己随便new。

    阅读开源库的源码的好处：学习如何实现功能和如何设计的。可能不需要创新，但是需要多读多总结。

  2. React.createElement也是工厂模式。 第一个参数是必填，传入的是似HTML标签名称，eg: ul, li 第二个参数是选填，表示的是属性，eg: className 第三个参数是选填, 子节点，eg: 要显示的文本内容 他是个工厂函数，创建的其实是window的实例，

class Vnode(tag,attrs,children){
    // ...省略
}
// 工厂函数
React.createElement=function(tag,attrs,children){
    //....
    //省略一堆验证操作
    return new Vnode(tag,attrs,children);
}

不需要关心构造函数，jsx是语法糖
  1. Vue的异步组件 Vue的自定义组件Vue.component('eleName',{data,template...}) 1.组件名为eleName; 2.data 数据属性：写函数; 3.template 模板：写组件的内容（元素和触发的事件） Vue的异步组件
Vue.component('async-example', function (resolve, reject) {
  setTimeout(function () {
    // 向 `resolve` 回调传递组件定义
    resolve({
      template: '<div>I am async!</div>'
    })
  }, 1000)
})

如你所见，这个工厂函数会收到一个 resolve 回调，这个回调函数会在你从服务器得到组件定义的时候被调用。你也可以调用 reject(reason) 来表示加载失败。这里的 setTimeout 是为了演示用的，如何获取组件取决于你自己。

Vue.component(
  'async-webpack-example',
  // 这个 `import` 函数会返回一个 `Promise` 对象。
  () => import('./my-async-component')

# 设计原则验证

  • 构造函数和创建者分离（通过工厂方法把构造函数和用户隔离，用户无new）
  • 符合开放封闭原则
